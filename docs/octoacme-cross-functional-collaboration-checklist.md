# OctoAcme — Cross-Functional Collaboration Checklist

## Purpose
Ensure effective collaboration across roles and teams throughout the project lifecycle. Use this checklist to identify and address collaboration gaps early.

## When to use
- At project kickoff to establish collaboration patterns
- During sprint planning to ensure all necessary roles are engaged
- When onboarding new team members to clarify collaboration expectations
- During retrospectives to identify collaboration improvements

---

## Pre-Project Collaboration Setup

### Stakeholder Alignment
- [ ] All key stakeholders identified and documented
- [ ] Communication preferences and frequency agreed upon
- [ ] Decision-making authority and escalation paths clarified
- [ ] Business Analyst engaged to gather initial requirements

### Team Formation
- [ ] All required roles identified for the project scope
- [ ] Role assignments confirmed with availability
- [ ] Team members introduced and contact information shared
- [ ] Collaboration tools and channels established (Slack, Teams, etc.)

### Initial Handoffs
- [ ] Business requirements handed off from Business Analyst to Product Manager
- [ ] Technical constraints shared from Solution Architect to team
- [ ] UX Designer briefed on user personas and goals
- [ ] Technical Writer informed of documentation expectations

---

## Planning Phase Collaboration

### Requirements Clarity
- [ ] Product Manager has reviewed requirements with Business Analyst
- [ ] Acceptance criteria reviewed by QA and Developers
- [ ] UX designs reviewed and approved by Product Manager
- [ ] Technical approach validated by Solution Architect

### Cross-Team Dependencies
- [ ] Dependencies on other teams identified and communicated
- [ ] Integration points discussed with relevant teams
- [ ] Shared timelines agreed upon for dependent deliverables
- [ ] Risk owners assigned for cross-team risks

### Ceremony Preparation
- [ ] Scrum Master has scheduled all agile ceremonies
- [ ] All required attendees confirmed for ceremonies
- [ ] Meeting agendas and preparation materials shared
- [ ] Remote collaboration tools tested (if hybrid/remote team)

---

## Execution Phase Collaboration

### Daily Collaboration
- [ ] Daily standups include all active contributors
- [ ] Blockers are raised and assigned owners immediately
- [ ] Cross-functional questions are routed to appropriate experts
- [ ] Scrum Master tracking and addressing impediments

### Design & Development Handoffs
- [ ] UX Designer available for implementation questions
- [ ] Design specifications accessible to Developers
- [ ] Code reviews include relevant architectural feedback
- [ ] Solution Architect consulted for significant technical decisions

### Quality Assurance
- [ ] QA engaged early in feature development
- [ ] Test cases reviewed by Product Manager for coverage
- [ ] Defects triaged with appropriate priority and owners
- [ ] Developers and QA collaborating on defect resolution

### Documentation
- [ ] Technical Writer informed of upcoming features
- [ ] Developers providing technical context for documentation
- [ ] Documentation review scheduled before release
- [ ] Release notes drafted and reviewed

---

## Release Phase Collaboration

### Pre-Release Coordination
- [ ] All roles aware of release timeline
- [ ] Release readiness reviewed with PM, QA, and Developers
- [ ] Stakeholders informed of release scope and timing
- [ ] Support team briefed on new features and changes

### Release Execution
- [ ] Clear ownership for release activities
- [ ] Communication channels monitored during deployment
- [ ] Rollback plan understood by all relevant parties
- [ ] Post-release verification responsibilities assigned

### Post-Release
- [ ] Release notes published and communicated
- [ ] Documentation updated and accessible
- [ ] Stakeholder notification sent
- [ ] Support team equipped to handle inquiries

---

## Retrospective Collaboration

### Feedback Collection
- [ ] All roles invited to retrospective
- [ ] Anonymous feedback option available if needed
- [ ] Cross-functional collaboration explicitly discussed
- [ ] Scrum Master facilitating inclusive discussion

### Action Items
- [ ] Action items assigned clear owners
- [ ] Cross-team action items communicated to relevant teams
- [ ] Collaboration improvement actions prioritized
- [ ] Follow-up scheduled for outstanding items

---

## Communication Best Practices

### Regular Touchpoints
| Touchpoint | Frequency | Participants | Purpose |
|------------|-----------|--------------|---------|
| Daily Standup | Daily | Developers, QA, Scrum Master | Progress, blockers |
| PM-PdM Sync | Weekly | PM, PdM | Alignment on priorities |
| Design Review | As needed | UX, PdM, Developers | Design approval |
| Architecture Review | As needed | Solution Architect, Developers | Technical guidance |
| Stakeholder Update | Weekly/Bi-weekly | PM, Stakeholders | Status and decisions |
| Retrospective | End of sprint | All team members | Continuous improvement |

### Communication Channels
- **Urgent issues:** Direct message or call
- **Blockers:** Daily standup + immediate escalation
- **Questions:** Team channel or direct message to expert
- **Status updates:** Project board and weekly reports
- **Decisions:** Documented in decision log

### Handoff Guidelines
1. **Document the handoff:** Ensure all necessary context is captured
2. **Confirm receipt:** Verify the receiving party has what they need
3. **Provide contact:** Share how to reach you for follow-up questions
4. **Set expectations:** Clarify timeline and next steps
5. **Follow up:** Check back to ensure smooth transition

## Role-Specific Collaboration Tips

### For Developers
- Reach out to UX Designer early if design questions arise
- Include QA in technical discussions about testability
- Consult Solution Architect before major technical decisions
- Provide Technical Writer with context for documentation

### For Product Managers
- Involve Business Analyst in requirements refinement
- Review designs with UX Designer before development starts
- Keep Stakeholders informed of trade-offs and decisions
- Ensure QA has clear acceptance criteria

### For Scrum Masters
- Create space for all voices in ceremonies
- Identify and remove impediments proactively
- Facilitate cross-functional problem-solving
- Coach team on effective collaboration practices

### For QA
- Engage early in feature planning for testability input
- Collaborate with Developers on defect understanding
- Communicate quality risks to Project Manager
- Share testing insights with Technical Writer
